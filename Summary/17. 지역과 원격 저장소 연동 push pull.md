# 지역과 원격 저장소 연동
## 1. 개인 접근 토큰 생성
**개인 접근 토큰 인증 개요**
> * 비밀번호 인증 대신 사용하는 강화딘 인증 방법
> * 깃허브 계정마다 토큰 생성 필요

**생성 방법**
> * 로그인 후 우측 깃허브 우측 메뉴 Settings
> * 좌측 하단 Developers settings
> * Genersate new token
> * 암호 인증
> * Generate token
> * 범위 선택

## 2. 깃허브 원격 저장소 수정 후 pull
**깃 깃허브 Push Pull**
> * Push: 원격 저장소로 올리기
> * Pull: 원격 저장소에서 로컬 저장소롤 내리기
>  > * 위에 둘다 커밋을 완료 후 진행을 해야함

**Pull**
> * 원격 저장소의 수정을 지역 저장소에 반영

## 3. 지역에서 원격 저장소로 Push
**지역 저장소에서 Push**
> * 쓰기 권한이 있어야 push 가능
>  > * 저장소에서 collaborator로 등록하면 다른 사람도 push 가능
> * Push: 로컬 저장소에서 남겨놓은 코드 변경 이력들이 원격 저장소롤 전송
> * 명령
>```
>$ git push [저장소 별칭] [브랜치 이름]
>
>## 옵션 -u를 사용하여 저장소명과 브랜치명을 입력하고 그 이후에는 인자 생략 가능
>$ git push -u [저장소명] [브랜치이름]
>
>## 항상 현재 브랜치를 기준으로 git push 명령어가 작동하면 매우 편리
>## 어느 브랜치에서 작업을 하든 git push만 날리면 원격 저장소에 동일한 브랜치로 업로드
>$ git config --global push.default current
>```

**git pull = git fetch + git merge**
> * fetch명령과 병합하는 merge 명령이 순차적으로 진행
>  > * fetch: 원격 저장소의 정보를 로컬 저장소로 가져오는 명령
>  > * merge: 변경된 정보를 로컬 저장소의 내용과 병합
