# 오픈소프트웨어 개요
## 1. 오픈소프트웨어 개요
**오픈소스 소프트웨어**
> * OSS 오픈소스 소프트웨어
>  > *  누구나 제한없이 코드를 보고 사용 가능한 오픈소스 라이선스를 만족하는 SW
> * OSI (Open Source Initiative)
>  > * 공개 소스 정의(OSD)의 관리 및 촉진을 담당하는 비영리 조합
> * 의미: 소프트웨어의 소스코드를 자유롭게 읽고, 수정, 재배포 가능
> * 자유 소프트웨어 개념에서 발전됨
> * 오픈소스 지원 관리 서버
>  > * github, gitlab, bitbucket
>  > * 위에 커뮤니티에서 개발된 아이디어와 소프트웨어를 교환
> * 장점
>  > * 누구든 보다 쉽게 연구하여, 새로운 프로그래밍 기술을 개발
>  > * 소프트웨어를 원하는 요구사항에 맞게 커스터마이징과 혁신 지원
> * 단점
>  > * 공개의 의무
>  > * 품질보증 및 유지보수, 보안 등의 어려움

## 2. 오픈소스 소프트웨어 저작권
**GNU General Public(GPL)**
> * 자유소프트웨어재단(FSS)에서 만든 라이선스
> * 프로그램은 목적이나 형태의 제한없이 사용이 가능
> * 프로그램을 이후 수정하고 배포하는 모든 경우에 무조건 GPL로 공개를 해야 함

**AGPL(GNU - Affero GPL)**
> * GPL 기반 라이선스
> * 네트워크 상호 작용하는 소프트웨어의 소스코드도 공개해야 한다는 조항을 추가한 라이선스
> * 서버에서 프로그램을 실행해서 다른 사용자들과 통신 중

**LGPL(GNU - Lesser GPL)**
> * GPL 라이선스를 사용할 때 소스코드를 다시 GPL로 공개해야 하는 부담 때문에 만들어진 라이선스
>  > * 전체 소스코드가 아닌 사용된 오픈소스 라이브러리에 대한 소스코드만 공개

**Apache License**
> * 소스코드 공개에 대한 의무사항은 라이센스에 포함되어 있지 않음

**MIT License**
> * MIT에서 학생들을 지원하기 위해서 만든 라이선스
> * 라이선스와 저작권 관련 명시 의무
> * 가장 느슨한 조건을 가지고 있어서 많은사람들이 사용하기 용이
