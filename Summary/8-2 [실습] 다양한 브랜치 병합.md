# [실습] 다양한 브랜치 병합
## 1. fast-forward 병합
**병합을 실습할 기본 상태**
> * 저장소 생성
> * 일렬 상태의 두 브랜치 생성

**저장소의 복사 활용**
> * 처음에 만든 저장소를 복사
> ```
> $ cp -r [저장소] [새로운 저장소 이름]
> ```
> * 복사한 저장소로 이동
> * 주요 브랜치로 이동
> * 병합 진행
> ```
> $ git merge [머지할 브랜치]
>
> ## 확인하기
> $ git lgagl
> ```

**fast-forward 병합 이전으로 돌아가기**
> * 두개의 reset 명령으로 가능
> ```
> $ git reset --hard HEAD~~
> $ git reset --hard 3d00e27d
> ```

**옵션 --ff-only**
> * 일렬 상태에서는 --ff-only로 빨리 감기 가능
> ```
> $ git merge [머지할 브랜치] --ff-only
> ```

## 2. 3-way와 squash 병합
**3-way병합**
> * 일렬 상태에서 3-way 병합
> ```
> $ git merge [머지할 브랜치] --no-ff
>
> ## 머지 메세지 입력하는 3-way 병합
> $ git merge [머지할 브랜치] --no-ff -m [메세지]
> ```

**다시 일렬 상태로 복구**
> * reset 명령으로 가능
> ```
> $ git reset --hard
> ```

**Squash 병합**
> * 새로운 커밋에 상대 브랜치 내용을 모두 합해 새로운 커밋 하나로 병합
>  > * 직접 메세지와 함께 커밋을 해야 함
>  > ```
>  > $ git merge --squash [머지할 브랜치]
>  > $ git commit -m [커밋 메세지]
>  > ```
> * squash 병합 이전으로 돌아가기
> ```
> $ git reset --hard HEAD~
> ```
