# 병합 충돌과 해결
## 1. 병합 충돌
**병합 충돌(conflict) 이해**
> * 병합을 진행 중 conflict 메세지 출력 -> 충돌 발생
> * 충돌의 기준
>  > * 수정되지 않거나 한쪽 브랜치에서만 수정되어서 발생
>  > * 두 브랜치 모두에서 변경 사항이 달라 발생

**충돌 해결**
> * 직접 파일 내용을 수정 후 저장
> * add, commit 진행
> * 필요하면 합병된 이전 브랜치 삭제

**충돌한 파일 내부 파악**
> ```
> <<<<<<<HEAD
> 현재 브랜치 HEAD의 수정 내용
> ===========
> 병합되는 브랜치의 수정내용
> >>>>>>>브랜치
> ```

## 2. 충돌 해결
**병합 취소 후 다시 병합**
> * 병합 취소
> ```
> $ git merge --abort
> ```
> * 다시 병합
> ```
> $ git merge [브랜치명]
> ```

**직접 수정 후 저장**
> * 충돌 표시 제거
> * git add, commit 다시 진행
