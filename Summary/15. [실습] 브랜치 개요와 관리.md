# [실습] 브랜치 개요와 관리
## 1. 브랜치 개요
**기본 브랜치**
>```
>## 기본 브랜치 이름 설정
>$ git config --global init.defaultBranch main
>
>## 이미 생성된 브랜치 이름 수정
>$ git branch -M [브랜치]
>```
> * main 브랜치는 자동으로 가장 마직막 커밋을 가리킴
> ```
> ## 단순 생성
> $ git branch [브랜치 이름]
>
> ## 브랜치를 생성하고 이동
> $ git switch -c [브랜치 이름]
> $ git checkout -b [브랜치 이름]
>
> ##브랜치 이동
> $ git switch [브랜치]
> $ git checkout [브랜치]
>
> ## 커밋이 발생한 브랜치 목록
> $ git branch
>
> ## 브랜치마다 마지막 커밋ID와 메세지도 함께 표시
> $ git branch -v
>```

## 2. 브랜치 관리
**최신 커밋 이전에서 두 브랜치 생성**
>```
>$ git switch main
>$ git checkout HEAD~
>$ git switch -c [브랜치]
>$ git checkout -b [브랜치]
>```

**브랜치 삭제**
>```
>$ git branh -d [브랜치]
>```

**브랜치 관련 도움말 보기**
>```
>$ git branch -h
>```
