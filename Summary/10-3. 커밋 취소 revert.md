# 커밋 취소 revert
## 1. 커밋 취소 revert
**커밋 취소**
> * revert: 지정한 특정 커밋을 취소해 바로 이전 상태로 되돌리는 방법
> * reset과 다르게 커밋할 것이 없고, 작업 영역이 깨끗해댜 수행가능

**revert 충돌과 해결**
> * 인자가 HEAD~와 같이 HEAD 이전을 취소하면 충돌 발생
> * 바로 이전 취소를 여러번 계속
> * 충돌 해결 방법
>  > * 파일수정
>  > * git add
>  > * git revert --continue

**명령 revert**
> * 옵션 --no-edit: 추가되는 커밋 메세지가 자동으로 revert 이전 커밋 메세지로 지정

**오류가 발생해 revert가 취소**
> * 커밋할 것이 있고 작업 영역이 깨끗 하지 않은면 오류발생
> * 해결 방법
>   > * 커밋을 하고 다시 revert하는 방법
>   > * 깃의 3영역 통일 시키기
>   > ```
>   > $ git reset --hard HEAD~
>   > ```

**바로 전 revert 취소**
> * reset을 통해 취소가 가능하다

## 2. reset VS revert 비교
> * reset: 지정된 이전 커밋으로 이동
> * revert: 지정된 이전 커밋을 취소하는 커밋이 생성
>  > * 이전 모든 이력은 그대로 남는다.
