# 버전관리 개요
## 1. 버전관리 이해
**버전 관리 필요성**
> * 지난 어느 시점의 파일, 소스의 내용을 확인하고 되돌리고 싶을 때 관리가 필요함

**버전 관리 개념**
> * 파일의 추가 및 수정 이력(추적) 관리
> * 파일의 과거 어느 시점으로 이동

**깃 도구 방식**
> * CLI(Command Line Interface): 텍스트로 명령을 입력하고 결과도 텍스트로 표시되는 인터페이스 방식
>   > * 윈도의 명령 프롬프트나 유닉스의 쉘이 대표적
> * GUI(Graphic User Interface): 윈도처럼 그래픽 대화 화면에서 마우스와 텍스트의 입력방식으로 명령을 입력하고 결과가 표시된는 인터페이스 방식

## 2. 커밋과 버전 관리
**버전 관리의 커밋 commit**
> * 저장소의 내부 상태를 사간별로 스냅샷 사진을 찍는 것처럼 모습을 기록하는 작업

**포인터 HEAD**
> * 가장 최근의 커밋을 가리키는 포인터

**이력을 저장하는 저장소**
> * 저장소(Git repository): 파일이나 폴더를 저장해 두는 곳
>   > * 파일이 변경 이력 별로 구분되어 저장
> * 원격 저장소(Remote Repository), 지역 저장소(Local Repository)
>   > * 원격 저장소: 파일이 원격 저장소에서 관리되며 여러 사람이 공유하기 위한 저장소
>   > * 지역 저장소: 내 PC에 파일이 저장되는 개인 저장소

**저장소 명령**
> * Clone: 원격 저장소를 지역 저장소에 복제
> * Push: 지역 저장소에서 원격 저장소로 올리기
> * Pull: 원격 저장소에서 지역 저장소로 내리기
> * add: 현재 파일을 작업 디렉토리에서 커밋 전 스테이징 영역에 추가
> * commit: 스테이징 영역에서 리포지토리에 추가

**버전관리시스템**
> * Git: 가장 많이 활용되는 분산 버전 관리 시스템
> * 다른 버전관리시스템: CVS, SVN, GIT, Mercurial, Bazaar
