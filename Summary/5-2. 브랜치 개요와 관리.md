# 브랜치 개요와 관리
## 1. 브랜치 개요
**브랜치**
> * 브랜치: 나무의 가지, 지점
> * 깃 브랜치: 버전관리를 수행하던 파일 집합을 복사해 독립적으로 개발하는 개념
> * 브랜치 병합(merge)
>  > * 독립된 브랜치에서 마음대로 소스 코드를 변경하여 원래 버전과 합치는 것
> * 브랜치 사용 장점
>  > * 저장소에서 다른 브랜치에는 영향없이 새로운 기능 개발, 버그 수정 가능

**기본 브랜치**
> * 기본 브랜치: 저장소 생성 시 처음 만들어지는 브랜치 (main)
> * 기본 브랜치 이름 설정
> ```
> $ git config --global init.defaultBranch main
>
> ## 이미 생성된 저장소의 브랜치 이름 수정
> $ git branch -M newBname
> ```

**브랜치와 HEAD**
> * 브랜치: 커밋을 가리키는 포인터
> * HEAD: 작업중인 브랜치의 최신 커밋을 가리키는 포인터
>  > * HEAD -> main -> 최신 커밋

**브랜치 생성**
> * 새로운 브랜치 생성
> ```
> ## 단순히 생성
> $ git branch [브랜치 이름]
>
> ## 생성하고 새 브랜치로 HEAD 이동도 수행
> $ git switch -c [브랜치 이름]
> $ git checkout -b [브랜치 이름]
> ```
> * 브랜치 확인
> ```
> ## 커밋이 발생한 브랜치 목록 보이기
> $ git branch
>
> ## 브랜치마다 마지막 커밋 ID와 메세지도 함께 표시
> $ git branch -v
> ```
> * 브랜치 이동
> ```
> $ git switch [브랜치]
> $ git checkout[브랜치]
>
> ## HEAD를 이전 브랜치로 이동
> $ git switch -
> $ git checkout -
>
> ## HEAD가 현재 브랜치가 아닌 그 이전 커밋을 가리키는 상태
> $ git checkout HEAD~
> $ git checkout [이전 커밋 아이디]
> ```

## 2. 브랜치 관리
**브랜치 생성과 이동**
>```
>## 브랜치 생성하고 이동
>$ git switch -c [브랜치 이름]
>$ git checkout -b [브랜치 이름]
>
>## 병합된 브랜치 삭제
>$ git branch [-d|--delete] [브랜치 이름]
>
>## 병합되지 않은 브랜치 삭제
>$ git branch -D [브랜치 이름]
>```

**브랜치 목록 보기**
>```
>## 현재 작업 브랜치를 기준으로 병합된 브랜치 목록 표시
>$ git branch --merged
>
>## 현재 작업 브랜치를 기준으로 병합되지 않은 브랜치 목록 표시
>$ git branch --no-merged
>
>## 인자인 브랜치 기준으로 병합된 브랜치 목록 표시
>$ git branch --merged [브랜치]
>
>## 인자인 브랜치 기준으로 병합되지 않은 브랜치 목록 표시
>$ git branch --no-merged [브랜치]
>```
