# 깃과 깃허브 개요
## 1. 깃 개요
**깃 Git**
> * 2005년, 리누스 토발즈가 개발
> * 대표적인 OSS 프로젝트
> * 장점: 모든 개발자는 지역 시스템에 코드의 전체 사본을 소유

**깃 기능**
> * 정의: 파일의 변경을 추적하는 버전관리 시스템
> * 기능
>   >* 여러 개발자가 함께 작업
>   >* 소스 코드의 변경 사항을 추적하는 데 사용
>   >* 소스 코드 관리에 분산 버전 제어 도구가 사용
>   >* 여러 개의 평행 분기를 통해 비선형 개발을 지원

**깃의 내부 주요 영역 구조**
> * working directory: 프로젝트 디렉토리(파일)
> * staging area: git add한 파일들이 존재하는 영역
> * git repository: working directory의 변경 이력들의 저장되어 있는 영역
> * stash: 임시 저장소

**깃에서 상태 간의 이동 명령**
> * add: working directory -> staging area
> * commit: staging area -> local repository
> * push: local reapository -> remote repository

**브랜치**
> * 나뭇가지(분기, 가지) 개념
> * 하나의 코드 관리 흐름

## 2. 깃허브 이해
**깃허브**
> * 버전 관리를 위한 서버 저장소 및 프로젝트 개발을 위한 협업 관리 서비스
