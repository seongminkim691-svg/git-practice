# 버전과 태그 활용(실습)
## 1. 버전과 태그 개요
**실제 SW 버전**
> * 버전(version): 프로그램을 수정, 개선할 때마다 코드를 구분하기 위해 부여된 식별자
>  > * 보통 두자리 또는 세자리 형태의 숫자
>  > * Git SW 버전: 2.39.0

**SemVer(Semantic Version)**
> * major.minor.patch: 세 자리 숫자 형태로 표기하는 버전
>  > * 메이저 번호
>  > * 마이너 번호: 메이저 버전에서 기능을 추가하거나 변경 사항이 있을 때 바꿈
>  > * 패치 번호: 버그 수정등 미미한 변화가 있을 때 바꿈

**태그(tag)**
> * 주석 태그: 태그 이름 + 정보(태그 작성자 메일, 태그 시각, 태그 메세지) 포함
> * 일반 태그: 태그 이름만 포함

## 2. 태그 명령
**주석 태그 생성**
>```
>## 작성한 사람의 이메일, 날짜, 메세지 등 정보 포함
>$ git tag -a v1.0.0 -m 'first version'
>
>## 기본 설정된 편집기로 메세지 편집
>$ git tag -a v1.0.0
>
>## 특정 커밋에 태그를 붙임
>$ git tag -a v1.1.0 [커밋아이디]
>$ git tag v1.1.0 HEAD~
>```

**일반 태그 생성**
> * 태그의 버전 정보만 관리
>```
>## 태그 버전 이름 중복 불가능
>$ git tag v1.0.1
>```
> * -a -m 사용 불가능

**태그 확인하기**
> * 태그 목록보기(예전 태그부터 표시)
>```
>$ git tag
>```
> * 태그 정보 보이기
>```
>$ git show v1.0.0
>```
> * 태그 삭제
>```
>$ git tag -d v1.0.0
>```
