# 다양한 브랜치 병합
## 1. 브랜치 병합 개요
**브랜치 병합**
> * 병합(merge): 두개의 브랜치를 하나로 모으는 과정
>  > * fast-forward(빨리 가기) 병합
>  > * 3-way 병합

**fast-forwar병합의 이해**
> * 현재 브랜치 main이 병합할 대상 커밋의 직접적이 뿌리(조상)가 되는 경우
>  > * 두 브랜치가 일렬 상태
> * 합치는 내용이 없어 main브랜치 이동을 한다.

**3-way 병합의 이해**
> * 두 분기가 갈라진 상태(두 브랜치의 조상이 같은 경우)
> * 새로운 커밋을 사용하여 두 기록을 합침
> * 병합을 진행하면 두 브랜치를 합쳐서 새로운 병합 커밋 생성

## 2. 병합의 다양한 옵션
**non fast-forward병합 옵션을 지정**
> * 병합 실행 시에 fast-forward병합이 가능하더라도 3-way 병합을 수행
> ```
> $ git merge --no-ff [병합할 브랜치]
> ```
> * 상태가 fast-forward인 일렬 상태에서만 병합 진행
> ```
> $ git merge --ff-only [병합할 브랜치]
> * 현재 브랜치에 병합 대상과의 합치는 커밋을 하나 생성해 병합
>   > * 커밋 이력과 병합되는 브랜치 이력도 남기지 않음
>   > ```
>   > $ git merge --squash [병합할 브랜치]
>   > ```
