# 깃 커밋과 로그
## 1. 버전관리를 위한 add, commit명령
**깃 영역**
> * working directory: 탐색기 상의 폴더 하부
> * staging area: 저장소의 .git 폴더의 파일 index
> * git repository: 저장소의 .git 폴더의 여러 정보 

**파일 생성후 add, commit**
> * 파일을 working tree -> staging area로 이동(복사)
> ```
> $ git add
> ```
> * staging area에 준비된 파일 -> git repository로 이동
> ```
> $ git commit
> ```

**깃 상태 보기**
> ```
> $ git status
> ```

**스테이징 영역에서 내용삭제(이전 내용으로 수정)**
>```
>$ git rm --cached [파일명]
>```

**커밋**
>* 커밋: 버전 관리를 위해 스테이지 영역의 내용에 대해 스냅샷 찍는 명령
>```
>$ git commit -m [커밋 메세지]
>```

## 2. 버전 로그 이력 확인
**커밋 로그 이력 보기**
>```
>$ git log
>
>## 로그 이력을 한 줄로 표시
>$ git log --oneline
>
>##로그 이력과 함께 파일의 변화 표시
>$ git log [--patch|-p]
>```
> * 모든 커밋은 구분하기 위해 커밋ID(체크섬)이 존재 -> 16진수로 구성

**커밋 정보 확인**
>```
>$ git show
>
>## 커밋 로그 한 줄과 파일 차이 표시
>$ git show --oneline
>
>## 파일 차이는 표시되지 않음
>$ git show -s
>
>## 지정한 HEAD의 커밋 정보 표시
>$ git show [HEAD]
>
>## 지정한 커밋아이디의 커밋 정보 표시
>$ git show [커밋아이디]
>```
